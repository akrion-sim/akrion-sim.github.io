Akrion · reinforcement learning

How the soccer AI actually learns

It plays games against itself, measures whether each decision beat expectations, and nudges its network weights toward the ones that did. That's policy-gradient learning — not tree search. Here's the real machinery: the equations, and how a single weight moves.

01Two networks make every decision

Each player reads the field vector s — positions and velocities of all players and the ball — and two neural nets turn it into a move and a judgement.

Actor  πθ(a | s)

weights θ · the policy

Outputs a probability for every action — pass, shoot, dribble, move, pick a gear. At play time it does one forward pass and samples. This is the thing that gets better.

Critic  Vφ(s)

weights φ · the value estimate

Estimates how good a state is — the expected future reward. It's the yardstick the actor is measured against: did an action do better or worse than the critic expected?

In the 5-a-side engine this is MAPPO-CTDE: the actor sees one player's local view (decentralised), while the critic sees the global state (centralised) — so its value estimate can account for things a single player can't see.

02The loop: play → measure → nudge

No equation is invented per game. The same objective is re-evaluated on each fresh batch of self-played experience, and the weights step a little each time.

A toy policy over five actions. Each cycle it samples one action, measures the advantage (did it beat the critic's baseline?), then nudges that action's probability up or down — clipped, so no single game moves it too far. Watch the genuinely-better actions drift up.

Step 1 — play. Sample actions from πθ, step the physics sim, and record (st, at, rt) at every tick. This is just collecting experience — no lookahead.

Step 2 — measure the advantage. How much better was an action than the critic predicted? First the one-step surprise, then a discounted sum of surprises (GAE):

δt = rt + γ·V(st+1) − V(st)
TD error — the reward plus what we now think the next state is worth, minus what we thought this state was worth. Positive = pleasant surprise. γ = 0.995 discounts the future one tick at a time.
Ât = Σk (γλ)k · δt+k   λ = 0.95
Advantage (GAE) — smooth the surprises over the rest of the play. Ât > 0 means "that action did better than expected here," which is exactly the signal we want to reinforce.

Step 3 — nudge the actor with the clipped PPO objective:

ρt = πθ(at|st) ⁄ πθold(at|st)
Probability ratio — how much the update has changed this action's probability from the policy that collected the data.
L(θ) = 𝔼[ min( ρtÂt, clip(ρt, 1−ε, 1+ε)·Ât ) ]  ε = 0.2
The whole point of PPO. Push up the probability of positive-advantage actions, push down the negative ones — but the clip caps how far one update can move, so a single lucky (or unlucky) game can't wreck the policy. Stable, near-monotone improvement.
θ ← θ + α·∇θL   α = 3·10−4
Gradient ascent on that objective — the actual weight update, repeated for 4 epochs over minibatches of 1024 transitions.

The critic is fit by plain regression toward the returns it should have predicted:

LV(φ) = 𝔼[ (Vφ(st) − Gt)² ]   φ ← φ − α·∇φLV
Squared error between the critic's guess and the observed return Gt = Σ γkrt+k. A better yardstick → cleaner advantages → a better actor.

03How one weight actually moves

The net is just matrix multiplies and nonlinearities. Backpropagation applies the chain rule to get ∂L/∂w for every weight; each steps by −α·∂L/∂w.

Intuitively, each weight moves in proportion to (the advantage) × (how much that weight raised the log-probability of the action we took). A weight that made a good action more likely gets reinforced; one that pushed a bad action gets turned down. Millions of these tiny, advantage-weighted nudges — over thousands of self-played games — are the entire learning process.

04Two things our system adds

Same weight-update math above; these only change the reward and the opponent.

r′t = rt + γ·Φ(st+1) − Φ(st)
Potential-based reward shaping. Φ is a hint (e.g. "closer to a good shooting spot"). Because it's a difference of a potential, it telescopes to zero over any loop — it can't be farmed and doesn't change the optimal policy, it just makes the gradient signal denser so learning is faster.

Self-play. The opponent is a frozen past champion. Playing it generates the trajectories; a challenger is promoted only when it beats the champion head-to-head on goals. The weights still change by the exact PPO math above — self-play only changes who you play, and guarantees the frontier only ever ratchets on genuine, unfakeable improvement.

05"Isn't it MCTS, since we simulate?"

No — and it's the same word for two different things. Simulating games is how we gather data; it is not how we pick moves.

 Rollouts  (what our learning uses)MCTS  (tree search)
PurposeCollect experience to compute gradientsSearch hypothetical futures to choose an action now
WhenDuring training, offlineAt decision time, per move
OutputA batch of (s, a, r) for the PPO updateOne chosen action, from a search tree
At play timeOne forward pass, then sampleHundreds of simulated branches
The 5-a-side is pure model-free PPO — it plays games to gather data, then does gradient steps. It never builds a search tree. The 11-a-side is a hybrid: the same actor-critic learning, plus model-based planning that does simulate forward — MPC (optimise a short control horizon) and a DP rollout teacher. Neural MCTS is available but was off in the current run. So even the big engine isn't "essentially MCTS" — it's PPO-style learning with optional planning bolted on.

In one line: we learn by policy gradients on self-played experience — play, measure advantage, nudge weights via clipped PPO. Simulation gathers the data; it doesn't pick the moves.
